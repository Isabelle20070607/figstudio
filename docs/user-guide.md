# FigStudio User Guide / 用户指南

## Script Workflow / 脚本工作流

Add a controlled block to the plotting area of your script:

在脚本的绘图区加入受控代码块：

```python
import figstudio

# data processing above this line
figstudio.open(locals(), script_path=__file__, block_id="main")

# figstudio:start main
# figstudio:end main
```

Open the editor, create or adjust layers, then click **Save code**. FigStudio replaces only the code between the matching markers.

打开编辑器，创建或调整图层，然后点击 **Save code**。FigStudio 只替换匹配 marker 中间的代码。

## Notebook Workflow / Notebook 工作流

Omit `script_path`:

不要传入 `script_path`：

```python
import figstudio

session = figstudio.open(locals())
```

The editor still generates code, previews figures, and exports files. **Save code** returns a replacement cell instead of modifying a notebook file.

编辑器仍会生成代码、预览图形和导出文件。**Save code** 会返回替换 cell 代码，而不会修改 notebook 文件。

## Mapping Data / 映射数据

Use **Variables** to choose source data and **Create layer** to map channels:

使用 **Variables** 选择数据源，并在 **Create layer** 中映射通道：

- DataFrame columns: choose the same DataFrame for X and Y, then select columns.
- Independent arrays or lists: choose `index` for X or choose another variable as X source.
- Error bars: choose `errorbar`, then optionally select an error source.
- Heatmap and contour: use a 2D ndarray or a gridded value source when possible.

- DataFrame 列：X 和 Y 选择同一个 DataFrame，再选择列。
- 独立数组或列表：X 可以选择 `index`，也可以选择另一个变量作为 X source。
- Error bars：选择 `errorbar`，再按需选择误差数据源。
- Heatmap 和 contour：尽量使用 2D ndarray 或网格化 value source。

## Editing Figures / 编辑图形

The editor has three main areas:

编辑器包含三个主要区域：

- Left: variables and layer creation.
- Center: Matplotlib preview and generated code.
- Right: figure, axes, subplot, and layer controls.

- 左侧：变量和图层创建。
- 中间：Matplotlib 预览和生成代码。
- 右侧：图、坐标轴、subplot 和图层控件。

Publish mode exposes publication-oriented controls such as font family and constrained layout. Explore mode keeps the surface shorter for quick iteration.

Publish 模式显示面向发表的控件，例如字体族和 constrained layout。Explore 模式保留更短的界面，适合快速迭代。

## Exporting / 导出

Use the PNG, SVG, or PDF buttons in the preview toolbar. Exports are generated by Matplotlib with the current `FigureSpec`.

使用预览工具栏里的 PNG、SVG 或 PDF 按钮。导出由 Matplotlib 根据当前 `FigureSpec` 生成。

## Existing Matplotlib Figures / 已有 Matplotlib Figure

You can pass an existing figure:

可以传入已有 figure：

```python
figstudio.open(locals(), figure=fig)
```

The beta inspects supported line artists and recreates editable generated layers. Unsupported artists remain best-effort metadata and should be treated as read-only context.

beta 会检查可支持的 line artist，并重建可编辑的生成图层。不支持的 artist 只作为 best-effort 元数据，应视为只读上下文。

## Common Errors / 常见错误

- `render_failed`: generated Matplotlib code could not run, often because data dimensions or selected columns do not match.
- `export_failed`: export rendering failed or an explicit output path could not be written.
- `writeback_failed`: the controlled block was missing, duplicated, nested, or used a different `block_id`.
- `writeback_io_failed`: the target script path could not be read or written by Python.

- `render_failed`：生成的 Matplotlib 代码无法运行，常见原因是数据维度或所选列不匹配。
- `export_failed`：导出渲染失败，或明确指定的输出路径无法写入。
- `writeback_failed`：受控代码块缺失、重复、嵌套，或 `block_id` 不匹配。
- `writeback_io_failed`：Python 无法读取或写入目标脚本路径。

## Recovery / 恢复方式

If a render fails, adjust the layer source mapping first. If writeback fails, verify the marker pair exactly matches the session `block_id`. Generated code is still shown in the code panel, so script or notebook work can continue manually.

如果渲染失败，先调整图层数据映射。如果写回失败，确认 marker pair 与会话 `block_id` 完全匹配。代码面板仍会显示生成代码，因此脚本或 Notebook 工作可以继续手动进行。
